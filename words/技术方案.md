# 技术方案文档

## 1. 技术选型说明

### 1.1 为什么选择 Next.js？
- **静态导出**: 支持 `output: 'export'`，适合部署到任何静态托管平台
- **App Router**: 现代化的路由系统，支持布局、加载状态等
- **性能优化**: 内置图片优化、代码分割、预渲染
- **生态丰富**: 大量插件和集成方案

### 1.2 为什么选择 Tailwind CSS？
- **开发效率**: 原子类样式，无需写 CSS 文件
- **主题定制**: 轻松自定义设计系统
- **响应式**: 内置断点系统
- **Dark Mode**: 支持 class 模式切换

### 1.3 为什么选择 Framer Motion？
- **React 友好**: 声明式动画 API
- **性能优秀**: 使用 GPU 加速
- **功能丰富**: 支持手势、滚动触发、布局动画

### 1.4 为什么选择 shadcn/ui？
- **可定制性强**: 代码完全可控，非黑盒组件
- **无障碍支持**: 基于 Radix UI，内置 ARIA 支持
- **TypeScript 优先**: 完整的类型定义
- **Tailwind 集成**: 与 Tailwind CSS 无缝配合

## 4. 数据管理方案

### 4.1 Skills 数据架构

```typescript
// 核心数据结构
interface Skill {
  id: string;
  title: string;
  description: string;
  category: "practical" | "github" | "community";
  tags: string[];
  githubUrl?: string;
  author?: string;
  stars?: number;
  installCommand?: string;
  content: string;
  icon: string;
  color: string;
  // 时间维度管理
  addedAt: string;
  updatedAt: string;
  isArchived?: boolean;
  weekNumber?: number;
  year?: number;
}

// 周归档结构
interface GitHubTrendingWeek {
  weekNumber: number;
  year: number;
  startDate: string;
  endDate: string;
  skills: Skill[];
}
```

### 4.2 数据分层

```typescript
// 1. 实战技能 - 长期维护
export const practicalSkills: Skill[] = [...];

// 2. 当前周 GitHub 热门 - 每周更新
export const currentGitHubTrending: Skill[] = [...];

// 3. 往期热门归档 - 历史数据
export const archivedGitHubTrending: GitHubTrendingWeek[] = [...];
```

### 4.3 数据更新机制

**自动更新流程**:
1. GitHub Actions 每周一 UTC 00:00 触发
2. 运行 `scripts/update-github-trending.ts`
3. 抓取 GitHub Trending 数据
4. 将上周数据归档到 `archivedGitHubTrending`
5. 更新 `currentGitHubTrending`
6. 提交并推送更改

## 7. 性能优化策略

### 7.1 构建优化
- 使用静态导出 `output: 'export'`
- 图片优化 `unoptimized: true` (静态导出需要)
- 代码分割 (Next.js 自动处理)
- Tree Shaking (自动移除未使用代码)

### 7.2 运行时优化
- 组件懒加载 (React.lazy + Suspense)
- 图片懒加载 (Next.js Image 组件)
- 动画性能优化 (使用 transform 和 opacity)
- 虚拟列表 (长列表时使用)

### 7.3 加载优化
- 首屏关键 CSS 内联
- 字体预加载
- 资源压缩 (Gzip/Brotli)
- CDN 部署

### 7.4 缓存策略
```javascript
// next.config.mjs

      {
        source: '/:all*(svg|jpg|png)',
```

### 8.2 GitHub Pages 部署
```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: cd frontend && npm ci
      - run: cd frontend && npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./frontend/dist
```
